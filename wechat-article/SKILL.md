# wechat-article

用于微信公众号文章的选题、标题、结构、排版、配图、转化话术和微信生态传播优化。适合生成技术文章、产品复盘、活动推文、个人 IP 长文和企业品牌内容。

## 核心规则

- 标题要清楚说明对象和收益；因为公众号打开率依赖订阅流中的快速判断。
- 标题不要堆叠夸张词；因为“震惊”“必看”会降低专业可信度。
- 摘要要补充标题未说完的信息；因为摘要会影响转发卡片点击。
- 开头三段必须进入问题；因为微信长文读者耐心有限。
- 正文段落控制在 2 到 4 行；因为手机阅读长段会压迫感强。
- 每个小标题只表达一个层级观点；因为微信读者常扫读。
- 技术文章要给代码、截图或清单；因为纯观点难以收藏。
- 产品文章要写清适用对象；因为泛泛宣传无法转化。
- 配图要服务理解，不要只放装饰图；因为微信图文加载和阅读节奏都受图片影响。
- 引流话术要自然嵌入上下文；因为硬广会提高取关风险。
- 结尾 CTA 只保留一个主动作；因为多个动作会稀释转化。
- 关键词要自然出现在标题、摘要、小标题和前 200 字；因为微信搜一搜会参考这些位置。
- 不要使用过多加粗和颜色；因为排版花哨会显得廉价。
- 列表项要并列同构；因为结构一致更容易阅读和收藏。
- 引用外部数据要标明来源；因为公众号内容会被读者二次传播。
- 涉及医疗、金融、教育承诺要保守表述；因为平台审核和合规风险高。
- 企业文章避免“领导视角”套话；因为读者关心问题解决，不关心组织自夸。
- 文章要设计可转发金句；因为朋友圈传播依赖短句表达立场。
- 不要把 README 原样搬成公众号；因为微信阅读需要叙事和节奏。
- 代码块要短，并解释关键行；因为移动端横向滚动体验差。
- 图注要说明图的意义；因为读者可能只看图和标题。
- 文章末尾可放“延伸阅读”，但不要超过 3 条；因为过多链接会分散注意力。
- 转载和引用要保留授权信息；因为版权投诉会影响账号。
- 长文要在 1500 到 3500 字内控制主线；因为过长需要系列化处理。

## 反例

```md
标题：重磅！AI 时代开发者必须知道的超级干货

问题：夸张但不具体，读者不知道内容解决什么问题。
```

```md
在当今数字化浪潮席卷全球的背景下，人工智能技术正在以前所未有的速度改变各行各业。

问题：套话开头，三段内没有进入正文。
```

## 正例

```md
标题：别让 AI 乱写代码：一份适合中文团队的 Skill 模板

摘要：我们把命名、提交、支付、小程序和内容规范拆成 8 个可复用 Skill，让 Agent 先读规则再动手。

开头：
最近我们在项目里发现一个反复出现的问题：AI 写代码很快，但经常不按团队习惯来。

比如接口命名混用英文缩写，提交信息写成“update”，小程序里直接把大对象塞进 setData。每个问题都不大，但 review 成本会被一点点抬高。
```

## 配置建议

- 输出默认包含：标题 5 个、摘要、正文、配图建议、结尾 CTA、SEO 关键词。
- 技术文默认加“可复制清单”和“常见误区”。
- 企业内容默认区分“对外发布版”和“内部复盘版”。
